# AI×わたし 相性診断ツール — Claude Code 制作指示書

## 0\. このドキュメントの使い方

このファイルの内容をそのままClaude Codeに貼り付けて「このドキュメントの内容でWebアプリを作ってください」と伝えてください。不明点があればClaude Codeが質問してくるので、都度このドキュメントを参照しながら回答してください。

---

## 1\. 概要・目的

- **名称**：AI×わたし 相性診断  
- **目的**：北九州の女性リーダー向けセミナー(2026/7/11)で配布し、参加者が「自分に合うAI活用法」を楽しく発見できる診断ツール。セミナー後もMe-Cherishコミュニティで継続利用する。  
- **成功基準**：  
  - セミナー参加者の大半が最後まで診断を完了する  
  - SNSへのシェアが生まれる  
  - まゆさんが後から回答傾向を見て、コンテンツ化・営業活動に活かせる

---

## 2\. 技術スタック(提案)

- **フロントエンド**：Next.js(App Router)+ TypeScript \+ Tailwind CSS  
- **データベース**：Supabase(無料枠で十分。PostgreSQL＋簡単なAPIが使える)  
- **ホスティング**：Vercel(まゆさんがme-cherish.comで使用済みのため親和性が高い)  
- **ドメイン**：me-cherish.comとは別の新規ドメイン、またはVercelの無料サブドメイン(例：ai-aisho-shindan.vercel.app)からスタートし、後日独自ドメインを追加してもよい  
- **画像生成(結果シェアカード)**：`html-to-image` または `@vercel/og` を使ってブラウザ上で結果をPNG画像化  
- **QRコード生成**：`qrcode` パッケージで配布用URLをQR化  
- **使用モデル**：Claude Fable 5(無料期間が7/7までのため、今週中に本番の質を作り切るのに向いている)。以降のメンテナンスはSonnet 5で問題ない

**注記(自由記述の扱い)**：アンケートの自由記述を自動分析する機能は、Claude APIをサーバー側から呼び出す必要がありSupabase連携より複雑になるため、今回のMVPには含めない。代わりに、診断結果内の「コピペ用プロンプト例」で、参加者自身が持ち帰ってChatGPT/Claudeに自由記述を貼って分析する使い方を案内する形にする。

Claude Codeへの指示例：

Next.js(App Router)とSupabaseを使って、診断アプリを新規プロジェクトとして作成してください。デプロイはVercelを想定します。

## 3\. 情報設計(質問リスト・最終版)

以下の8問構成にする。

1. **職場でAIを使うことについて**(permission)  
2. **普段のお仕事に一番近いもの**(domain)  
3. **今のAI活用レベル**(usage、0〜3のスコア)  
4. **AIに対する今の気持ち**(feeling、0〜3のスコア)  
5. **普段の仕事で一番時間がかかっている・大変なこと**(priority) — 資料・提案書づくり／情報収集・リサーチ／メール・チャット対応／会議準備・議事録まとめ／SNS発信・画像づくり／エクセルなどのデータ入力・集計／アンケート作成・集計分析  
6. **普段使っているツール環境**(platform) — Google系／Microsoft系／両方／こだわりなし  
7. **自分の得意なこと**(strength)  
8. **ニックネーム(任意入力)**(nickname)

質問文・選択肢の正確な文言は、添付の`AIxわたし相性診断.html`(リファレンス実装)をそのまま踏襲すること。

## 4\. 診断結果ロジック・コンテンツ

タイプ判定ロジック、各タイプの文言(強み／伸びしろ／おすすめツール)、`priority`×`platform`に応じた具体的なツール提案は、すべて添付の`AIxわたし相性診断.html`の実装をそのまま移植する。追加で以下の要素を必ず含めること：

### コピペ用プロンプト例

`priority`の回答ごとに、その場でChatGPT/Claudeに貼って使えるプロンプト例を表示し、「コピー」ボタンで clipboard にコピーできるようにする(内容はリファレンス実装の`promptExamples`と同じ)。特に「資料・提案書づくり」は、①見出し構成だけ作らせる→②確認後に肉付けする、という2段階のプロンプトにする(いきなり完成文章を求めると質が下がるため)。

### AIを「秘書」にする3つのコツ(全タイプ共通で表示)

①繰り返す指示はカスタム指示・メモリ機能に登録する②背景情報を先に伝える③最終判断は自分で行う

### マスターしていくロードマップ(Lv1〜Lv3)

Lv1「まずは1つのAIに話しかけてみる」→Lv2「テンプレ化して使い回す」→Lv3「Claude(相談・下書き)／Claude Cowork(資料横断・複数アプリ連携)／Claude Code(繰り返し作業の自動化)を使い分けて仕組み化する」という3段階を表示する。

### マスコットキャラクター

リファレンス実装のSVGマスコット(ローズカラーの耳を持つ丸顔のキャラクター)を、スタート画面・結果画面に踏襲する。ふきだしメッセージ付きで表示すること。

## 4\. 診断結果ロジック

既存の6タイプ判定ロジックを維持：

comfort \= usage \+ feeling

comfort \<= 1 → タイプA「はじめの一歩さん」

comfort \>= 5 → タイプE「使いこなし発信者さん」

それ以外は domain に応じて：

  office → タイプB「効率化ハンター」

  creative → タイプC「発想ブースター」

  people, specialist → タイプD「信頼の伴走者」

  management → タイプF「経営ナビゲーター」

各タイプの文言(強み／伸びしろ／おすすめツール／今日からできる一歩)は既存の診断アプリ(`AIxわたし相性診断.html`として別途共有済み)からそのまま移植する。

### time\_sink × platform に応じた「具体的なツールの掛け合わせ」提案

`platform`の回答によって、Google系ツールかMicrosoft系ツールかを出し分ける。以下のロジックで組み合わせを提示：

| time\_sink | Google Workspace中心の場合 | Microsoft中心の場合 | 共通で使えるツール |
| :---- | :---- | :---- | :---- |
| 資料・提案書づくり | Gemini(Google Slides連携) | Copilot(PowerPoint連携) | Claude・ChatGPT(構成づくり)＋Canva(デザイン仕上げ) |
| 情報収集・リサーチ | Gemini(Google検索連携) | Copilot(Bing連携) | Claude・ChatGPT(Deep Research)・Perplexity |
| メール・チャット対応 | Gemini(Gmail連携) | Copilot(Outlook連携) | ChatGPT・Claude(文面のトーン調整) |
| 会議準備・議事録まとめ | Gemini(Google Meet文字起こし) | Copilot(Teams文字起こし) | Otter.ai・Claude・Notion AI(要点整理) |
| SNS発信・画像づくり | — | — | Canva(AI機能)・ChatGPT/Claude(キャプション)・CapCut(動画) |
| データ入力・集計 | Gemini(スプレッドシート) | Copilot(Excel) | ChatGPT(データ分析機能) |

「浮く時間の目安」も一言添える(数値は目安として明記)：

- 資料・提案書づくり → 「1本あたり30分〜1時間の時短」  
- 情報収集・リサーチ → 「調べ物1回あたり15〜30分の時短」  
- メール・チャット対応 → 「1日あたり10〜20分の時短」  
- 会議準備・議事録まとめ → 「会議1回あたり20〜30分の時短」  
- SNS発信・画像づくり → 「投稿1本あたり20〜40分の時短」  
- データ入力・集計 → 「集計作業1回あたり15〜30分の時短」

## 5\. 画面構成(ユーザーフロー)

1. **スタート画面**：タイトル・一言説明・「はじめる」ボタン(ニックネーム入力欄も任意でここに設置)  
2. **質問画面(全8問)**：1問ずつ表示、進捗バー、戻るボタンあり  
3. **結果画面**：  
   - タイプ名・キャッチコピー  
   - 強み／伸びしろ  
   - おすすめツール(タイプ別の一般的な内容)  
   - 時間がかかっている業務に対する具体的な掛け合わせ提案＋浮く時間の目安  
   - 「この会場での自分の位置」(同じイベントタグの回答者の中で同じタイプの割合。件数が少ない場合は非表示にする分岐を入れる)  
   - 結果シェア用の画像ダウンロードボタン  
   - LINE公式アカウント登録・Instagramフォローへの導線ボタン(クリックしたら`wants_followup`をtrueで記録)  
   - 「もう一度診断する」ボタン

## 6\. データモデル(Supabase テーブル設計)

テーブル名：`diagnosis_responses`

| カラム名 | 型 | 説明 |
| :---- | :---- | :---- |
| id | uuid (primary key, default gen\_random\_uuid()) |  |
| created\_at | timestamptz (default now()) |  |
| event\_tag | text | どのイベント/場面での回答か(例：`kitakyushu_seminar_20260711`、`mecherish_community`など)。URLのクエリパラメータ`?event=`で受け取る |
| nickname | text (nullable) |  |
| permission | text |  |
| domain | text |  |
| usage\_level | int |  |
| feeling\_level | int |  |
| time\_sink | text |  |
| strength | text |  |
| platform | text |  |
| result\_type | text | A〜F |
| wants\_followup | boolean (default false) |  |

**重要**：個人情報(氏名・連絡先など)は保存しない。ニックネームは任意のニックネームであり本名の入力を求めない設計にする。

## 7\. 管理者用の集計ダッシュボード

- パス：`/admin`(URLを知っている人だけがアクセスできる形でよいが、簡易パスコード入力による保護を追加する。環境変数`ADMIN_PASSCODE`と比較する簡易実装でよい)  
- 表示内容：  
  - `event_tag`で絞り込みできるフィルター(セミナー用データとコミュニティ用データを分けて見られるように)  
  - 総回答数  
  - タイプ別(A〜F)の件数・割合(棒グラフ)  
  - `time_sink`別の件数(棒グラフ)  
  - `platform`別の件数  
  - `wants_followup`がtrueの件数・割合  
- グラフライブラリは`recharts`を使用

## 9\. 追加機能の実装優先度

**MVP(セミナーまでに必須)**

1. 8問の診断フロー  
2. 結果表示ロジック(タイプ判定＋ツール掛け合わせ＋時短目安)  
3. Supabaseへのデータ保存(event\_tag対応)  
4. LINE・Instagram導線ボタン  
5. QRコード生成(配布用URL)

**あれば良い(時間があれば)** 6\. 結果シェア画像のダウンロード機能 7\. 「会場での自分の位置」表示(同一event\_tag内の集計) 8\. 管理者ダッシュボード(セミナー当日までに間に合わなければ、セミナー後でも可)

Claude Codeには「まずMVPを動く状態で仕上げて、その後追加機能を実装してください」と伝えると進めやすい。
